# GeoGaga — Server Flavor

**GeoGaga - Server Flavor** — это автоматически собираемые файлы данных `geoip.dat` и `geosite.dat`, специально оптимизированные для настройки умной **серверной** маршрутизации трафика. Сборка предназначена для использования непосредственно на стороне заграничного VPS-сервера под управлением Xray/Sing-box в связке с популярными панелями управления (**3x-ui**, **Remnawave**, **Marzban** и др.).

Главная особенность данной серверной версии — агрегация, очистка от дубликатов и перепаковка специфичных категорий в две ключевые группы правил с префиксом `GEOGAGA-` (приведены к верхнему регистру для совместимости со всеми версиями ядер), а также сохранение мировых баз данных **v2fly** и Loyalsoldier в режиме `as-is`. Это позволяет серверу эффективно отсекать мусорный трафик, маскировать запросы к заблокированным в РФ хостингам и хостить классические правила.

---

## 🛠 Рекомендуемый порядок правил на сервере

Для достижения максимального быстродействия, экономии трафика VPS и правильной логики распределения ресурсов рекомендуется выстраивать правила маршрутизации на сервере в следующем порядке:

1. **Блокировка (`Block`)** ➡️ Сюда отправляется реклама, телеметрия ОС, трекеры и тяжелый торрент-трафик. Соединение сбрасывается моментально (`blackhole`), не нагружая внешний канал сервера.
2. **Маскировка / Прокси (`Proxy Upstream`)** ➡️ Трафик к заблокированным ресурсам, белым спискам и зарубежным хостинг-провайдерам. Перенаправляется в интерфейс маскировки (например, через **Cloudflare WARP** или каскадный прокси) для обхода блокировок со стороны зарубежных сервисов.
3. **Напрямую (`Direct`)** ➡️ Финальное правило (По умолчанию). Весь остальной трафик, не попавший в предыдущие списки, должен идти напрямую через основной сетевой интерфейс вашей VPS.

### Пример структуры в конфигурации Xray JSON:
```json
"routing": {
  "domainStrategy": "AsIs",
  "rules": [
    {
      "type": "field",
      "outboundTag": "block_blackhole",
      "domain": ["geosite:GEOGAGA-BLOCK"],
      "ip": ["geoip:GEOGAGA-BLOCK"]
    },
    {
      "type": "field",
      "outboundTag": "warp_outbound",
      "domain": ["geosite:GEOGAGA-PROXY"],
      "ip": ["geoip:GEOGAGA-PROXY"]
    },
    {
      "type": "field",
      "outboundTag": "direct",
      "network": "tcp,udp"
    }
  ]
}
```

## 📦 Описание категорий GeoSite (`geosite.dat`)

Категория `geosite.dat` содержит доменные имена, ключевые слова и регулярные выражения, распределенные по целевым группам:

### 🚫 `geosite:GEOGAGA-BLOCK`
Предназначена для жесткой блокировки рекламы, трекеров, телеметрии и P2P-трафика прямо на сервере.
* **Включает в себя категории:**
  * Из репозитория *hydraponique/roscomvpn-geosite*:
    * `torrent` — популярные торрент-трекеры и анонсеры для исключения паразитной P2P-нагрузки на сервер.
    * `private`, `category-ads` — приватные диапазоны и базовый отфильтрованный список рекламных доменов.
  * Из репозитория *runetfreedom/russia-blocked-geosite*:
    * `category-ads-all` — расширенная база рекламных сетей и систем аналитики.
    * `geosite:win-spy` — домены сбора телеметрии ОС Windows.

### 🔵 `geosite:GEOGAGA-PROXY`
Список доменов, требующих маскировки (например, пуска через Cloudflare WARP для обхода блокировок со стороны зарубежных сайтов, отклоняющих запросы с хостинг-провайдеров).
* **Включает в себя категории:**
  * Из репозитория *hydraponique/roscomvpn-geosite*:
    * `whitelist`, `category-ru` — списки доменов общего обхода, требующих проксирования/обработки на сервере.

### 🌐 Оригинальные теги (Добавлено AS-IS)
* Полная база доменов из репозитория **v2fly/domain-list-community** (`dlc.dat`). Все категории (`google`, `netflix`, `apple`, `github` и т.д.) импортируются без изменений для возможности создания классических точечных правил на основе базы **v2fly**.

## 🌐 Описание категорий GeoIP (`geoip.dat`)

Категория `geoip.dat` оперирует массивами IP-адресов и CIDR-подсетей (IPv4 и IPv6).

### 🚫 `geoip:GEOGAGA-BLOCK`
Сетевые диапазоны, подлежащие моментальному сбросу на внешнем интерфейсе сервера.
* **Включает в себя категории:**
  * Из репозитория *hydraponique/roscomvpn-geoip*:
    * `private` — диапазоны частных адресов (RFC 1918) и локальных путей для фильтрации на сетевом уровне.

### 🔵 `geoip:GEOGAGA-PROXY`
Сетевые диапазоны и подсети операторов, перенаправляемые в интерфейс маскировки трафика.
* **Включает в себя категории:**
  * Из репозитория *hydraponique/roscomvpn-geoip*:
    * `direct` — подсети российских операторов связи и локальных автономных систем.
    * `whitelist` — доверенные IP-адреса критической инфраструктуры.

### 🌐 Оригинальные теги (Добавлено AS-IS)
* Полная база IP-адресов мировых зон из репозитория **Loyalsoldier/v2ray-rules-dat**. Все категории (`cn`, `telegram`, `google` и т.д.) импортируются без изменений для сохранения полной совместимости с классической маршрутизацией.

## 🔄 Автоматическое обновление

Сборка полностью автономна и обновляется раз в сутки с помощью **GitHub Actions** в **05:55 (GMT+3)**. В процессе работы скрипта все исходные `.dat` файлы декомпилируются через Protobuf-структуры, очищаются от дубликатов, оптимизируются (включая проверку вхождения поддоменов в родительские зоны и пересечения IP-диапазонов) и упаковываются обратно в бинарный формат Protobuf. Имена кастомных категорий строго переводятся в верхний регистр (`UPPERCASE`) во избежание конфликтов синтаксиса Xray.
